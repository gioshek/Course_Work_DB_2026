# BookStreamDB Course Project

**BookStream** — курсовой проект по базам данных: онлайн-сервис цифровых книг с каталогом, покупками, подписками, балансом, отзывами, избранным, чтением книг, акциями/скидками и расширенной админ-панелью для работы с БД.

## 1. Стек проекта

- **БД:** Microsoft SQL Server
- **Backend:** Python, FastAPI, pyodbc
- **Frontend:** React, Vite, Axios
- **Документация API:** Swagger UI
- **ОС разработки:** Ubuntu

## 3. Подготовка базы данных

Перед запуском backend и frontend нужно создать и заполнить базу данных.

1. Запустить SQL Server.
2. Открыть проект в VS Code.
3. В расширении MSSQL подключиться к серверу.
4. Последовательно выполнить SQL-файлы из папки `database`:

```text
01_create_database.sql
02_create_tables.sql
03_insert_test_data.sql
04_views.sql
05_indexes.sql
06_procedures.sql
07_functions.sql
08_triggers.sql
09_test_queries.sql
10_set_demo_passwords.sql
```

Важно: файлы `11`, `12`, `13` больше отдельно не нужны. Логика акций, отчётов и удаления акций уже встроена в основные файлы `01–10`.

После выполнения SQL-файлов должна быть создана база:

```text
BookStreamDB
```

## 4. Настройка backend

Перейти в папку backend:

```bash
cd ~/CourseWorkeSpring2026/backend
```

Создать виртуальное окружение, если оно ещё не создано:

```bash
python3 -m venv venv
```

Активировать виртуальное окружение:

```bash
source venv/bin/activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

Проверить, что установлен драйвер SQL Server:

```bash
python3 -c "import pyodbc; print(pyodbc.drivers())"
```

В списке должен быть примерно такой драйвер:

```text
ODBC Driver 18 for SQL Server
```

Файл `backend/.env` должен содержать настройки подключения к БД. Пример:

```env
DB_DRIVER=ODBC Driver 18 for SQL Server
DB_SERVER=localhost
DB_NAME=BookStreamDB
DB_USER=sa
DB_PASSWORD=your_password
DB_TRUST_CERTIFICATE=yes
```

Запуск backend:

```bash
cd ~/CourseWorkeSpring2026/backend
source venv/bin/activate
uvicorn app.main:app --reload --host 127.0.0.1 --port 8000
```

Backend будет доступен по адресу:

```text
http://127.0.0.1:8000
```

Swagger-документация API:

```text
http://127.0.0.1:8000/docs
```

## 5. Настройка frontend

Открыть второй терминал и перейти в папку frontend:

```bash
cd ~/CourseWorkeSpring2026/frontend
```

Установить зависимости:

```bash
npm install
```

Запустить frontend:

```bash
npm run dev
```

Frontend будет доступен по адресу:

```text
http://localhost:5173
```

## 6. Тестовые пользователи

После запуска SQL-файла `10_set_demo_passwords.sql` можно использовать тестовые аккаунты:

```text
admin / admin123
```

```text
giorgi / 1234
```

`admin` — администратор сайта. Ему доступна админ-панель.

`giorgi` — обычный пользователь для проверки покупок, подписок, библиотеки, избранного и баланса.

## 7. Основной функционал пользователя

Пользователь может:

- просматривать каталог книг;
- искать книги по названию, автору, жанру или описанию;
- использовать нечёткий поиск с опечатками;
- смотреть страницу книги;
- покупать книги с внутреннего баланса;
- видеть скидки и финальную цену;
- пополнять баланс;
- оформлять подписку;
- добавлять книги в избранное;
- читать доступные книги;
- сохранять прогресс чтения;
- оставлять отзывы.

## 8. Основной функционал администратора

Администратор может:

- добавлять новые книги;
- добавлять авторов, жанры и издательства;
- загружать текст книги из `.txt`-файла;
- создавать акции и скидки;
- привязывать книги к акциям;
- удалять акции;
- смотреть метрики БД;
- формировать интерактивные отчёты;
- просматривать SQL-объекты проекта;
- просматривать журнал действий `AuditLog`.

## 9. Акции и скидки

В проекте используются таблицы:

```text
Promotion
BookPromotion
```

Акция содержит название, промокод, процент скидки, дату начала, дату окончания и статус активности.

Книга может быть привязана к акции. В каталоге и на странице книги отображаются старая цена, скидка и итоговая цена. При покупке списывается именно итоговая цена с учётом скидки.

## 10. Отчёты администратора

В админ-панели есть раздел `Отчёты`. Там администратор может формировать отчёты с фильтрами:

- продажи за период;
- продажи по книгам;
- продажи по пользователям;
- отчёт по книгам с фильтром по жанру, издательству, рейтингу и скидкам;
- отчёт по пользователям;
- отчёт по жанрам;
- отчёт по журналу действий.

Отчёты формируются через backend, который обращается к SQL-процедурам в базе данных.

## 11. SQL-объекты

В проекте используются:

- таблицы для хранения основных сущностей;
- представления для каталога, отзывов, платежей и отчётов;
- процедуры для пользовательских и админских операций;
- функции для расчёта скидок, итоговой цены, рейтинга и баланса;
- триггеры для логирования действий в `AuditLog`;
- индексы для ускорения поиска и связей.

В админ-панели есть отдельный раздел `SQL-объекты`, где можно посмотреть объекты БД и их назначение.

1. Открыть каталог книг.

3. Найти книгу поиском с опечаткой.
4. Открыть страницу книги.
5. Проверить отображение скидки.
6. Войти как обычный пользователь.
7. Пополнить баланс.
8. Купить книгу.
9. Открыть библиотеку.
10. Добавить книгу в избранное.
11. Оставить отзыв.
12. Войти как администратор.
13. Добавить книгу.
14. Создать акцию.
15. Привязать книгу к акции.
16. Сформировать отчёт.
17. Проверить `AuditLog`.

## 14. Краткое описание проекта

BookStream — это веб-приложение для онлайн-чтения цифровых книг. Проект демонстрирует работу с реляционной базой данных: хранение связанных сущностей, покупки, платежи, подписки, отзывы, избранное, прогресс чтения, акции, отчёты администратора, SQL-процедуры, функции, представления, индексы и триггеры.
